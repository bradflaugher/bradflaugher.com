$ Navigating the AI Agent Landscape

A practical guide to choosing AI agents based on real-world testing. Let's cut through the hype and focus on what actually works.

~/overview The Agent Landscape

AI agents are evolving rapidly. After testing dozens of options, here's what actually matters:

SECURITY = "Never run agents on your main system"
CAPABILITY = "Choose based on what they can actually do"
EXTENSIBILITY = "Look for agents that can connect to your tools"

~/agent_types Four Types of AI Agents

1. Research Agents

These agents search and synthesize information. But most don't actually browse the web—they query a pre-indexed cache. Fine for Wikipedia facts, useless for analyzing that article you just found.

2. GUI/Computer Use Agents

Agents like Google's Mariner or OpenAI's Operator click around your screen like a human. Impressive to watch, painful to use. They're slow, expensive, and usually the wrong tool for the job.

3. Terminal & Execution Agents

The powerhouses. These agents can write code, execute commands, browse the actual web, and get real work done. This is where the magic happens—but also where security becomes critical. Examples: Claude Code, Aider, Manus, OpenHands.

4. Multi-Agent Systems

Tools like MetaGPT that simulate entire teams. One agent plays PM, another architect, another developer. Powerful for complex projects, overkill for most tasks.

~/terminal_agents Terminal & Execution Agents: The Real Deal

These are the agents that actually get work done. They can execute commands, write code, and interact with systems. But with great power comes great responsibility—always run them in isolated environments.

Claude Code & Aider: Closed vs Open Coding Agents

Claude Code and Aider represent the closed/open source divide in terminal-based coding agents:

Claude Code (Closed Source):

• Anthropic's offering - Built directly by the Claude team

• Polished experience - Seamless integration with Claude's capabilities

• Research preview - Currently in limited availability

• Terminal focused - Pure command-line power for development

Aider (Open Source):

• Community driven - Fully open source and extensible

• Model agnostic - Works with Claude, GPT-4, and other LLMs

• Git integration - Native version control workflows

• Available now - Install and use immediately

Security for both: Always run in isolation:

• Raspberry Pi - Physical isolation on cheap hardware
• Virtual Machine - Easy to snapshot and restore
• Docker Container - Lightweight and disposable

Both are incredibly powerful for development tasks when properly sandboxed.

Manus & OpenHands: Closed vs Open Multi-Modal Agents

Manus and OpenHands showcase the closed/open source approach to multi-capability agents:

Manus (Closed Source):

• Swiss Army knife agent - Terminal + web + light GUI

• Built-in sandboxing - Every task runs in its own VM

• Live web browsing - Actually visits pages, not cached data

• Polished UX - Professional interface and workflows

OpenHands (Open Source):

• Community alternative - Fully transparent and modifiable

• Docker-first - Security through containerization

• LLM agnostic - Choose your preferred model provider

• Extensible - Add your own capabilities and integrations

Both offer powerful multi-modal capabilities with different trade-offs between polish and flexibility.

~/other_agents The Rest of the Landscape

Research Agents: Limited but Easy

Most "AI search" tools query cached data. They're fast and cheap but can't:

• Analyze specific links you provide
• Access real-time information
• Navigate non-indexed sites

# Use Manus if you need actual web browsing

GUI Agents: Niche Use Cases

Computer use agents have their place, but it's small:

Good for: QA testing, accessibility, legacy system interaction
Bad for: General development, research, anything with an API

Multi-Agent Systems: Enterprise Scale

MetaGPT and similar tools orchestrate multiple specialized agents. Great for large projects where you need structured workflows and documentation. Overkill for individual developers.

~/extensions Making Agents More Powerful: MCP

MCP (Model-Context Protocol) is how agents connect to external tools and services. It's the difference between a general chatbot and a specialized assistant.

What MCP Enables

• Connect to company databases and tools
• Integrate with third-party APIs
• Control IoT devices and hardware
• Access specialized domain knowledge

Building Safely with MCP

With MCP, agents can do almost anything—which is both powerful and dangerous. Always implement:

• Confirmation prompts for destructive actions

• Rate limiting to prevent runaway operations

• Audit logging for all agent actions

• Minimal permissions following principle of least privilege

The Golden Rule: Isolation Is Non-Negotiable

Never give an AI agent direct access to your main system.

Whether it's Claude Code on a Raspberry Pi, Aider in Docker, Manus in its VM, or OpenHands in containers—isolation isn't optional. The most powerful agents are the ones that can execute code, which makes sandboxing essential.

~/recommendations Practical Recommendations

For Pure Coding Tasks:
→ Claude Code (closed) - Polished Anthropic experience
→ Aider (open) - Flexible, git-integrated, available now
For Research + Development:
→ Manus (closed) - Best-in-class multi-modal capabilities
→ OpenHands (open) - Transparent and extensible alternative
For Large Projects:
→ MetaGPT for multi-agent orchestration
For GUI Automation:
→ Computer use agents (only when necessary)
Always Remember:
→ Sandbox everything that executes code
→ Look for MCP support for extensibility
→ Start small, expand gradually

~/conclusion Choose Wisely, Sandbox Always

The most capable agents are Terminal & Execution agents—they can actually do things, not just talk about them. But with that power comes risk.

Success with AI agents comes from:

1. Understanding capabilities - Know what each type can actually do

2. Prioritizing security - Always use isolated environments

3. Building incrementally - Start simple, add extensions as needed

4. Choosing open vs closed - Balance polish against flexibility for your needs

The future is bright for those who approach agents thoughtfully. Choose the right tool, sandbox religiously, and build step by step.
