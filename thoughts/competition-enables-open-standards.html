Why Competition Between Proprietary Platforms Enables Open Standards

October 19, 2025
A counterintuitive insight: competition between proprietary platforms can promote open standards better than consolidation around the "most open" option. Your strategic choices today determine what's possible tomorrow.

The Conventional Wisdom (and Why It's Wrong)

The conventional wisdom in the free software community goes something like this: "Always choose the most open platform available. If everyone uses the most open option, we'll collectively move toward freedom."

This sounds logical. It's also strategically naive.

The reality is more nuanced and counterintuitive: competition between proprietary platforms can promote open standards more effectively than consolidation around a "more open" platform. Understanding this dynamic is crucial for making strategic technology choices that actually advance software freedom in the long term.

The Gaming Industry: A Perfect Case Study

The Naive View

Steam Deck runs on Linux. Therefore, everyone should use Steam Deck to support Linux gaming. More Steam Deck users means more Linux gaming support. Simple, right?

The Sophisticated View

Competition between PlayStation, Xbox, and PC gaming forces developers to use cross-platform technologies. When these platforms compete, no single vendor can control the graphics API standard. Microsoft can't force everyone to use DirectX because Sony and Nintendo need alternatives. This keeps open graphics APIs like Vulkan and OpenGL maintained, relevant, and improving.

Here's what actually happens when you have three competing gaming platforms:

  • Cross-platform development becomes essential. Game studios want their titles on all platforms to maximize revenue. They use cross-platform engines like Unity and Unreal Engine.
  • Open APIs stay relevant. These engines rely on Vulkan and OpenGL because they need to target multiple platforms with different proprietary APIs.
  • Linux gaming benefits indirectly. The work done to support PlayStation's graphics stack (which uses Vulkan-like APIs) directly benefits Linux gaming.

The Consolidation Trap

Now imagine everyone consolidated on Steam Deck and Windows PC. What would happen?

  • DirectX becomes the de facto standard. Why support OpenGL or Vulkan if everyone's on Windows or Proton (which translates DirectX anyway)?
  • Cross-platform engines become less important. Developers target DirectX directly for better performance.
  • Open APIs stagnate. Without competitive pressure, Vulkan and OpenGL development slows.
  • Future alternatives become impossible. New platforms must support DirectX to be viable, giving Microsoft permanent control.

Paradoxically, using PlayStation (a proprietary platform) maintains the competitive pressure that keeps open graphics APIs alive, which ultimately benefits Linux gaming more than consolidating on Steam Deck would.

The FreeBSD Connection

PlayStation 5's operating system is based on FreeBSD. Sony's investment in PlayStation drives FreeBSD development, with improvements flowing back to the open source BSD ecosystem. If PlayStation didn't exist and everyone used Linux-based systems, this diversity and development investment would disappear. Competition between proprietary platforms built on different open source foundations strengthens the entire ecosystem.

Mobile: The Same Pattern

The GrapheneOS Trap

GrapheneOS appears to be the "most open" mobile option. It's based on AOSP, has better privacy than stock Android, and removes Google services by default. The conventional wisdom says everyone concerned about mobile freedom should use GrapheneOS.

But here's the problem: GrapheneOS still requires Google Pixel hardware, and approximately 90% of users end up installing Google Play Services anyway because critical apps (banking, work, transportation) simply don't function without them.

More importantly, if everyone in the free software community consolidated on GrapheneOS, what would happen?

  • Android becomes the only "open" mobile standard. Developers target Android APIs exclusively.
  • Progressive Web Apps (PWAs) become irrelevant. Why support web standards if everyone's on Android?
  • Future alternatives must be Android-compatible. New platforms can't compete without supporting Android apps.
  • Google wins forever. Even in the "free software" world, Google's APIs become the permanent standard.

The Alternative Strategy: Promote Open Standards

A more strategic approach: use any phone (iPhone or Android), but reject native apps entirely in favor of Progressive Web Apps and open web standards.

When iPhone and Android compete, neither can control the standard. Developers who want to reach both platforms need platform-agnostic solutions. The web fills that gap. By using PWAs exclusively, you:

  • Keep web standards relevant and maintained. Competition drives web API improvements.
  • Remain platform-agnostic. You're not locked into any vendor's ecosystem.
  • Enable future alternatives. When a truly free phone emerges, it just needs a good browser to run your apps.
  • Vote with your behavior. Every time you choose a PWA over a native app, you're promoting open standards.

This is more strategically aligned with software freedom than using GrapheneOS, even though GrapheneOS is technically "more open." You're challenging the root problem (closed app ecosystems) rather than accepting a slightly better version of it.

AI Agents: The Newest Example

The MCP Revolution

The AI agent space is experiencing this dynamic right now. Anthropic introduced the Model Context Protocol (MCP), an open standard for connecting AI applications to external systems. Think of it as "USB-C for AI agents."

Here's what's fascinating: MCP exists because of competition between AI providers. If everyone consolidated on a single AI platform (say, OpenAI), that platform would use proprietary APIs for tool integration. There would be no need for an open standard.

But because Claude, ChatGPT, Gemini, and others compete, developers need a platform-agnostic way to build tools and integrations. MCP fills that gap. Now we have open source AI agents like Crush by Charm Bracelet that can use MCP to integrate with any tool or data source, regardless of which AI model powers the agent.

What This Enables

MCP's existence means:

  • Open source agents are viable. Projects like Crush don't need to build proprietary integrations for every tool.
  • Tool developers write once, support everywhere. An MCP server works with any MCP-compatible agent.
  • No single AI vendor controls the ecosystem. You can switch between Claude, GPT, or local models without losing your tools.
  • Future AI platforms can compete immediately. New entrants just need to support MCP to access the entire ecosystem.

If the AI community had consolidated on "the most open option" (perhaps a single open source model), we likely wouldn't have MCP. The competitive pressure between proprietary AI providers created the need for an open standard, which now benefits everyone—including open source projects.

The General Principle

Competition → Cross-Platform Needs → Open Standards

When proprietary platforms compete, they cannot all use each other's proprietary APIs. This creates demand for open, cross-platform standards. These standards enable future alternatives and prevent any single vendor from achieving permanent control.

When Consolidation Kills Standards

Consolidating on "the most open option" can actually harm open standards when:

  1. It reduces competitive pressure. One dominant platform means no need for cross-platform standards.
  2. It entrenches one vendor's APIs. Even if more "open," those APIs become the de facto standard.
  3. It makes alternatives impossible. Future platforms must be compatible with the dominant platform's APIs.
  4. It reduces diversity. Monocultures are fragile; diversity enables resilience and innovation.

When Competition Promotes Standards

Competition between proprietary platforms promotes open standards when:

  1. No single vendor can control the standard. Multiple platforms need neutral ground.
  2. Cross-platform development becomes valuable. Developers want to reach all platforms.
  3. Open standards fill the gap. They become the bridge between competing platforms.
  4. Future alternatives remain possible. New platforms can use the same open standards.

Strategic Framework for Technology Choices

When Open Solutions Exist: Use Them

For desktop, laptop, and server computing, truly open solutions exist and work well:

  • Hardware: System76, Framework, Purism, or used ThinkPads with coreboot
  • Operating Systems: GNU/Linux distributions (Debian, Fedora, Arch, etc.)
  • Applications: Vast ecosystem of free software
  • Standards: Open formats, open protocols, open languages

Recommendation: Use 100% free software. There's no excuse not to.

When Open Solutions Don't Exist: Maintain Competition and Open Standards

For gaming, where no truly open console exists:

  • Don't consolidate on Steam Deck. This would kill open graphics APIs.
  • Use Linux gaming (Bazzite, etc.) where it works well. Support open platforms when viable.
  • Use PlayStation or other consoles where Linux doesn't work. Maintain competitive pressure.
  • Result: Competition keeps Vulkan/OpenGL relevant, enabling future Linux gaming improvements.

For mobile, where no truly open phone exists:

  • Don't consolidate on GrapheneOS. This would kill open web standards.
  • Use any phone (iPhone, Android, whatever). The OS doesn't matter strategically.
  • Reject native apps entirely; use PWAs only. Promote open web standards.
  • Result: Competition keeps web standards relevant, enabling future free phones.

For AI agents, where the landscape is rapidly evolving:

  • Support open standards like MCP. Use agents that support MCP regardless of underlying model.
  • Don't consolidate on one AI provider. Competition drives open standards.
  • Use open source agents when possible. Projects like Crush benefit from MCP's existence.
  • Result: No single AI vendor controls the ecosystem; alternatives remain possible.

The Asymmetry Principle

This framework aligns with Richard Stallman's "asymmetry principle" from the GNU philosophy: encourage movement toward open standards, not toward closed ecosystems.

Approach Direction Long-term Result
Consolidate on GrapheneOS Toward Android ecosystem Android becomes permanent standard
Use any phone + PWAs only Toward open web standards Enables future free alternatives
Consolidate on Steam Deck Toward DirectX (via Proton) DirectX becomes permanent standard
Use Linux + PlayStation Toward Vulkan/OpenGL Enables future Linux gaming
Consolidate on one AI provider Toward proprietary APIs One vendor controls ecosystem
Support MCP across providers Toward open standards Enables open source agents

Tactical vs. Strategic Thinking

The key insight is distinguishing between tactical improvements and strategic direction:

GrapheneOS is a tactical improvement (better privacy than stock Android) but a strategic failure (entrenches Android ecosystem).

Using PWAs on iPhone is a tactical compromise (proprietary OS) but a strategic success (promotes open web standards).

Steam Deck is a tactical improvement (Linux-based gaming) but a strategic risk (could entrench DirectX via Proton).

Using PlayStation alongside Linux gaming is a tactical compromise (proprietary console) but a strategic success (maintains open graphics APIs).

When open solutions don't exist, strategic direction matters more than tactical purity. Your choices today determine what's possible tomorrow.

Conclusion: Think Like a Strategist

The free software movement needs more strategic thinking and less tactical purity. When truly open solutions exist, use them without compromise. But when they don't exist yet, focus on maintaining the conditions that will enable them to emerge in the future.

This means:

  • Promoting open standards over "more open" platforms
  • Maintaining competitive pressure between proprietary platforms
  • Preventing consolidation that would entrench any single vendor's APIs
  • Thinking long-term about network effects and ecosystem dynamics

Competition between proprietary platforms isn't ideal, but it's often better than consolidation around a "more open" platform that would kill the open standards we need for true alternatives to emerge.

Your behavior today shapes what's possible tomorrow. Choose strategically.
